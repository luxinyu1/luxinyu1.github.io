快速排序

发表于 2020-04-12 分类于 算法

快速排序仍然采用了分治法的思想，只是在其基础上引入了哨兵，实现了跨度更大的交换。因此，主程序实现用递归显得更加简单和清晰。

很显然Partition是这个算法的精髓。

不过想一想，每次都初始化同一个哨兵位置是最合理的吗？显然不是，当分治时一边为0个元素，中间为哨兵，另一边为n-1个元素时，陷入了最坏的情况，与插入排序差不多慢，算法导论上有详细的说明。所以可以增加一点随机性去规避极端性：

快速排序在常数倍分割时时间复杂度为O(nlogn)，在最坏条件下（一边为0个元素，中间为哨兵，另一边为n-1个元素）时，时间复杂度为O(n^2)

数据库中的范式

发表于 2020-04-12 更新于 2020-04-17 分类于 数据库

1NF

1NF的定义为：符合1NF的关系中的每个属性都不可再分，即原子属性，这很简单。

2NF

2NF在1NF的基础之上，消除了非主属性对于码的部分函数依赖。

函数依赖：若在一张表中，在属性（或属性组）X的值确定的情况下，必定能确定属性Y的值，那么就可以说Y函数依赖于X，写作 X → Y，即X确定Y。

  1. 平凡函数依赖：确定的东西是原先条件的一部分，像(A, B) → A 或(A, B) → B这种，平凡函数依赖在任意条件下必然成立，所以称其为平凡。
  2. 非平凡函数依赖：确定的东西不是原先条件的一部分，即X不是Y的一部分，如(A, B) → C

部分函数依赖：假如Y函数依赖于X，即X确定Y，但同时Y并不完全函数依赖于X，即X的真子集可以确定Y，那么我们就称Y部分函数依赖于X，反之为完全函数依赖。

码：一个属性或属性组，若除K之外的所有属性都完全函数依赖于 K，那么我们称K为候选码，简称为码，码可以有多个，故可以选一个为主码。
例如，（学号、课名）这个属性组就可以是码。（姓名，课名）这个属性也可是码（只要没有重复的姓名）。

主键：等同于主属性。

主属性和非主属性：包含在任何一个码中的属性成为主属性，其余为非主属性。
在上面的例子里，学号和课名都是主属性。
假设有（学号，课名） → 系名，学号 → 系名，即存在非主属性系名对码（学号，课名）的部分函数依赖，不满足2NF

3NF

3NF在2NF的基础之上，消除了非主属性对于码的传递函数依赖。
学号 → 系名，同时 系名 → 系主任，就不符合3NF。

BCNF

在3NF的基础上消除主属性对于码的部分与传递函数依赖。

求属性集闭包的方法

关系模式R<U，F>，其中U={A，B，C，D，E}，F={AB→C，B→D，C→E，EC→B，AC→B}，求(AB)+

  1. 令X(0)=AB。
  2. 求X(1)。先列出X(0)的非空子集，即AB的非空子集为{A，B，AB}。然后扫描F集合，寻找{A，B，AB}可能存在的函数依赖，就可以得到：AB→C，B→D。于是就可以求得X(1)=X(0)∪C∪D=ABCD。然后判断X(0)如果等于X(1)就结束，所求即为答案，如果X(0)不等于X(1)就继续计算。
  3. 求X(2)。同第二步求X(1)得非空真子集，然后在F中一次寻找函数依赖，可以得到：AB→C，B→D，C→E，AC→B。求得X(2)=X(1)∪C∪D∪E∪B=ABCDE。这时候发现X(2)已经等于全部属性集U了，就结束计算，得出(AB)+ = ABCDE。

注：如果计算的X(2)不等于U,并且仍然不等于X(1)，则继续计算下去。直到满足X(i)=X(i-1)或X(i)=U时停止，此时所求闭包为X(i)。

简而言之，就是首先把要求闭包的元素求所有非空子集，对每一个非空子集在关系集F种进行扫描，得到可以确定的其他元素以后添加进来，继续递归求所有非空子集，得到可以确定的元素，直到两次确定的元素相同或已经把所有属性全部包括为止。

装配线问题

发表于 2020-04-12 分类于 算法

某个汽车工厂共有两条装配线,每条有$ n $个装配站。装配线$ i $的第$ j $个装配站表示为$ S_i[j] $,在该站的装配时间为$ a_i[j] $。一个汽车底盘进入工厂,然后进入装配线$ i $($ i $为 1 或 2),花费时间为$ e_i $。在通过一条线的第$ j $个装配站后,这个底盘来到任一条装配线的第$ j+1 $个装配站。如果它留在相同的装配线,则没有移动开销。但是,如果它移动到另一条线上,则花费时间为$ t_i[j] $。在离开一条装配线的第$ n $个装配站后,完成的汽车底盘花费时间为$ x_i $离开工厂。
待求解的问题是,确定应该在装配线 1 内选择哪些站,在装配线 2 内选择哪些站,才能使汽车通过工厂的总时间最短。

暴力办法由于每个站都有两个选择，总装配路线数就是$ 2^{n} $，当n很大时，变得不可行。

递归

对于每一个问题$ S_i[j] $，其子问题有两种情况：

  1. 经历了装配线的转换到了$ j $号装配站
  2. 没经历转换直接到了$ j $号装配站

所以转化成递归问题，就是两种情况：

  1. $ j=1, f^* = min(e_1+a_1[j],e_2+a_2[j]) $
  2. $ j\neq1, f_1[j] = min(f_1[j-1]+a_1[j],f_2[j-1]+t_2[j-1]+a_1[j]) $
    $ j\neq1, f_2[j] = min(f_2[j-1]+a_2[j],f_1[j-1]+t_1[j-1]+a_2[j])$

动态规划

用填表法，可以先将所有$ f_1[j],f_2[j] $的值全部计算出来，同时设置箭头指向用来回溯，或者新设一个$ l_i[j] $专门记录到$ S_i[j] $的最快路径的上一个工作台所在的装配线号。

问题

Professor Canty conjectures that there might exist some $ e_i $, $ a_i[j] $, and $ t_i[j] $ values for which FASTEST-WAY produces $ l_i[j] $ values such that $ l_1[j] = 2 $ and $ l_2[j] = 1 $ for some station number $ j $. Assuming that all transfer costs $ t_i[j] $ are nonnegative, show that the professor is wrong.

简而言之，就是要证明对于同一个$ j $，不可能出现$ l_1[j] = 2 $ and $ l_2[j] = 1 $这样的情况。

假设$ l_1[j] = 2 $ and $ l_2[j] = 1 $成立，则：

$ l_1[j] = 2 $ 即 $ f_2[j-1]+t_2[j-1] < f_1[j-1] ①$

$ l_2[j] = 1 $ 即 $ f_1[j-1]+t_1[j-1] < f_2[j-1] ②$

显然①②两个式子矛盾，故假设不成立。

最大公共子序列问题

发表于 2020-04-12 分类于 算法

穷举法：对于每一个$ X_m $的子序列，验证其是否为$ Y_m $的子序列。
假设$ X_m = {x_1, x_2, x_3, … , x_m} $有m个元素，它就有$ 2^{m} $个子序列。对于每个$ X_m $中的子序列，验证其是否也是$ Y_m $的子序列需要花费$ O(n) $的时间。
总时间复杂度就成了 $ O(n2^{m}) $，这是一个突破天际的复杂度。
那么就要考虑更好的算法了。

动态规划

子问题有两种情况：

  1. xm = ym，即两个序列最后一个元素相同，那么子序列就是Xm-1和Ym-1的子序列，子序列长度就是之前长度+1
  2. xm != ym，即两个子序列最后一个元素不同，那么子序列就是(子序列(Xm,Ym-1), 子序列(Xm-1,Ym))中长的那个，子序列长度也是原来两个长度大的那个

这样就很容易构造二维表来存放最大公共子序列的长度。例如CABADCDB和DCADBCDAB，填表规则是：

  1. 若xy方向两个元素一样，就将其左上方格子的值+1填入
  2. 若xy方向两个元素不一样，就取其上边一格和左边一格中值较大的填入
  3. E代表左上箭头，同时也意味着其处于子串中，L代表左箭头，U代表上箭头

由上图右下角E6出发，经过U5,E5,E4,U3,E3,E2,L1,L1,E1到达边界，得出最大公共子序列为CABCDB

动态规划

发表于 2020-04-12 分类于 算法

概述

动态规划的本质是对递归中产生的结果进行存储（用表存储，在代码中以数组的形式体现），避免重复调用相同子问题的递归造成时间上的损失。可以看成是分治法的一种改进。其与分治法的最大不同是：它遵循自底向上的计算方式，而分治法通常是自顶向下的。
学术一点来讲：动态规划是研究多阶段决策过程中采用最优化原理，把多阶段的问题转化为单阶段解决。

动态规划的基本步骤

  1. 找出最优解的性质并刻画其结构特征
  2. 递归定义最优值
  3. 以自底向上的方式计算最优值
  4. 根据计算最优值时的信息构造最优解

归并排序

发表于 2020-04-12 分类于 算法

归并排序本质上是一次自顶向下的二分分割，分割后的n个单元素子问题的两两合并的过程
类比同样是树形结构的二分查找的时间复杂度是O(logn)，加上一个合并的过程就成为了归并排序，其复杂度就应该是O(nlogn)
但是本质上划分并没有意义，因为可以将原来的数组就看成一个已经划分成单元素子问题的集合，所以简而言之：并不需要递归过程，直接可以从树的底部开始向上合并

矩阵连乘问题

发表于 2020-04-12 更新于 2020-04-17 分类于 算法

需要找到以最小次数计算矩阵乘法的运算方式，在矩阵连乘过程中，不同的“加括号”方式会对运算速度造成很大影响。

设$ A[i:j] $表示$ i $位置到$ j $位置的矩阵连乘，所需要的最少数乘次数为$ m[i:j] $，则原问题的最优值为$ m[1:n] $，$A_i$的维数为$ p_{i-1}p_i $。

当$ i=j $时，$ A[i,j]=Ai $，所以$ m[i,j]=0 $, $ i = 1,..,n $

当$ i \neq j $时，$ m[i,j]=min \lbrace m[i][k] + m[k+1][j] + p_{i-1}p_kp_j \rbrace $

动态规划填表的话，应该以对角线为基准，向右上逐对角线填写，右上角$ m[1][n] $即为答案。

大整数乘法

发表于 2020-04-12 分类于 算法

大整数乘法(n位2进制)X*Y可以采用归并的思想解决。
将X按位数划分成两半，高位为a，低位为b：
$$ X = a * 2^{(n/2)} + b $$
同样的，将Y按位数分成两半，高位为c，低位为d：
$$ Y = c * 2^{(n/2)} + d $$
这样$$ XY = ac * 2^{n} + (ad + bc) * 2(n/2) + bd $$
然而很容易发现，$ T(n) = 4T(n-1) + O(n) $ （这里的4为$ ac $ $ ad $ $ bc $ $ bd $四次$ T(n-1) $的乘法）时间复杂度没有改进！
那么优化一下：$ XY = ac * 2^{n} + ((a-b)(d-c) + ac + bd) * 2(n/2) + bd $，这样一来，表面上看起来更烦了，但相比起上面，这个式子只要计算$ (a-b)(d-c) $ $ ac $ $ bd $ 三次$ T(n-1) $的乘法，比上面的式子花的时间更少了。
为啥用$ (a-b)(d-c) $不用$ (a+b)(d+c) $呢？显然减比加的位数要少。

Transformer

发表于 2020-04-12 更新于 2020-04-21 分类于 NLP

Transformer是encoder-decoder模式神经网络的一个应用，属于无监督学习中的一种。

传统Word2Vector的不足

传统词向量化的过程中，不同语料中一个字/词的向量都是相对固定的，这样就忽略了上下文信息对词带来的影响。

Multi-head Attention

Attention机制

假设输入为一句英文的话：Tom chase Jerry.
如果用传统encoder-decoder模型，那么在翻译Jerry时，所有输入单词对翻译的影响都是相同的，但显然Jerry的贡献度应该更高。
引入attention后，每个单词都会有一个权重：$ (Tom,0.3)(Chase,0.2)(Jerry,0.5) $。
具体实施过程是：

  1. 引入三个需要训练的矩阵：$ Q $，$ K $，$ V $，其中$ Q $为当前要去查询的词矩阵，$ K $为被查询的词矩阵，$ V $为实际特征信息，也可以理解为未加权的特征
  2. 通过计算不同词$ i,j $之间的 $ q_i·k_j $，生成词$ i,j $相关性的评分$ Score $，$ Score $越高，词与词之间越相关
  3. 计算每一个字/词，与其他字/词的两两关系后，除以维度（因为维度越大并不代表越相关，大的维度反而会使分数异常的大）再$ softmax $一下，再去分别乘对应的$ V $，就是这个字在句子/段落中的重要性权值，即$$ Attention(Q,K,V)=softmax(\frac{QK^{T}}{\sqrt{d_k}})V $$

举个例子，在Thinking Machines这个词组中，当我们处理Thinking这个词时，我们需要计算句子中所有词与它的Attention Score，这就像将当前词作为搜索的query，去和句子中所有词（包含该词本身）的key去匹配，看看相关度有多高。我们用$ q_1 $代表Thinking对应的query vector，$ k_1 $及$ k_2 $分别代表Thinking以及Machines对应的key vector，则计算Thinking的attention score的时候我们需要计算 $q_1k_1$ 与 $q_1k_2$，同理，我们计算Machines的attention score的时候需要计算 $q_2k_1$ 与 $q_2k_2$，然后进行尺度缩放与softmax归一化。

这个时候，这个词/字的特征向量就带有整个词组/句子的特征信息啦！

Multi-headed机制(通常8个)

类似于卷积神经网络，多头机制：

  • 通过不同的head得到多个特征表达
  • 拼接特征
  • 全连接降维

在多头注意力机制中则是通过h个不同的线性变换对$ Q,K,V $进行投影，最后将不同的attention结果拼接起来：
即：$$ head_i = Attention(QW_i^{Q},KW_i^{K},VW_i^{V}) $$
$$ MultiHead(Q,K,V)=Concat(head_1,head_2,…,head_h) W^{O} $$

Multi-head Attention可以进行多次，效果更佳。

位置信息的表达、层归一化与残差连接

BERT用正弦余弦信号的组合表示了字/词在句子/段落中的相对位置信息。
BERT也使用了层规范化进行优化。
深层次网络是有很多网络层为冗余层的，那么我们希望这些冗余层能够完成恒等映射，保证经过该恒等层的输入和输出完全相同。

Transformer

Encoder

Encoder由N=6个相同的layer组成，layer指的就是上图左侧的单元，最左边有个”Nx”，这里是6个。每个Layer由两个sub-layer组成，分别是multi-head self-attention mechanism和fully connected feed-forward network。其中每个sub-layer都加了residual connection和normalisation，因此可以将sub-layer的输出表示为：
$$ SubLayerOutput=LayerNorm(x+SubLayer(x)) $$

Decoder

Decoder和Encoder的结构差不多，但是多了一个attention的sub-layer，这里先明确一下decoder的输入输出和解码过程：

  • 输出：对应i位置的输出词的概率分布
  • 输入：encoder的输出 & 对应i-1位置decoder的输出。所以中间的attention不是self-attention，它的K，V来自encoder，Q来自上一位置decoder的输出
    解码：这里要特别注意一下，编码可以并行计算，一次性全部encoding出来，但解码不是一次把所有序列解出来的，而是像RNN一样一个一个解出来的，因为要用上一个位置的输入当作attention的query。

明确了解码过程之后最上面的图就很好懂了，这里主要的不同就是新加的attention多加了一个mask，因为训练时的output都是ground truth，这样可以确保预测第i个位置时不会接触到未来的信息。

残缺棋盘问题

发表于 2020-04-12 分类于 算法

残缺棋盘是一个有2k*2k(k≥1)个方格的棋盘，其中恰有一个方格残缺。要对其用四种三格板（L形骨牌）进行覆盖。

用分治法解决挺容易的。显然每块大棋盘都可以划分成4块k*k个方块的小棋盘，8块(k/2)*(k/2)个方块的更小棋盘，以此类推……
但问题是，每次分治划分出来的棋盘只有一块是残缺的，另外三块并不是。那么就要给另外三块制造一些残缺性。
最简单的是：在非残缺棋盘的相邻拐角处放置一块三格版，这样每块小棋盘都拥有残缺性啦！
这样就满足分治法的要求了，大问题能分成一样的小问题，小问题的解也能拼凑成大问题的解。
